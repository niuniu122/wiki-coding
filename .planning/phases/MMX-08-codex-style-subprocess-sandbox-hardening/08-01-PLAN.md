# Plan 08-01: Independent Policy and Fail-Closed Launch Contract

## Objective

Carry an immutable sandbox policy from the core invocation decision to the process launcher, and make backend failure a typed zero-fallback result.

## Tasks

1. Add failing core/CLI tests for permission-to-sandbox mapping, restart behavior, repeated selection, and policy snapshotting across in-flight/parallel invocations.
2. Add `ToolSandboxPolicy` to the core effect/port contract and pass it through the built-in adapter only to process-backed tools.
3. Add failing launcher tests for unavailable/denied backends and a target side-effect proving restricted execution never retries directly.
4. Introduce typed launch errors, stable sandbox denial codes/receipts, and unsupported-platform fail-closed behavior while retaining direct execution for explicit disabled policy.

## Verification

- `cargo test -p minimax-core --locked tool`
- `cargo test -p minimax-cli --locked --test tool_loop`
- `cargo test -p minimax-tools --locked --test process_tools`
