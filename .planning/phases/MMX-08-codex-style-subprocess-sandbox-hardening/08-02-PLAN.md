# Plan 08-02: Linux Bubblewrap Enforcement and Malicious Canaries

## Objective

Implement a minimal Linux Bubblewrap filesystem/network boundary and prove it against hostile transitive Cargo build code.

## Tasks

1. Add failing pure argument/discovery tests for safe Bubblewrap resolution, minimal read-only mounts, private home/tmp, writable `/workspace`, protected metadata overlays, required namespace flags, seccomp instructions, and forbidden workspace-local backends.
2. Implement a dedicated Linux sandbox module with Bubblewrap, the raw cBPF syscall filter, runtime/toolchain/cache mounts without credential files, temporary-home lifetime, and existing process-tree cancellation integration.
3. Add a Linux adversarial Cargo build-script fixture that attempts a host-marker read, host-local TCP and Unix-domain connections, and a workspace write; first observe the old/direct bypass, then require restricted denial and disabled-policy access.
4. Add repeat/parallel/cancellation tests and make the hosted Linux job fail rather than skip when Bubblewrap security evidence is required.

## Verification

- `cargo test -p minimax-tools --locked sandbox`
- `cargo test -p minimax-tools --locked --test sandbox_adversarial`
- Linux hosted CI with `bwrap --version`
