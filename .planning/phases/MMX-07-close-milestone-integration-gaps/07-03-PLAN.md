# Plan 07-03: Complete Distribution and Evidence Binding

## Objective

Produce one installable artifact containing the Rust default and explicit TypeScript fallback, and make cutover evidence invalidate when critical product inputs change.

## Tasks

1. Stage a deterministic platform npm package under `target` with launcher, native binary, legacy `dist`, metadata, licenses, docs, and no source/planning/model weights.
2. Extend strict archive verification to validate and smoke-run the extracted Rust default and verify the legacy bin mapping/files.
3. Add cross-platform LF policy plus a deterministic cutover product fingerprint used by local gates and hosted evidence.
4. Update installation, rollback, support, and README documentation to the actual Rust-default architecture.

## Verification

- `npm run build && npm run build:rust:release && npm run package:rust && npm run verify:rust-release`
- `npm run test:launcher`
- negative package/fingerprint fixture tests

COMP-04, REL-01, REL-03, REL-04
