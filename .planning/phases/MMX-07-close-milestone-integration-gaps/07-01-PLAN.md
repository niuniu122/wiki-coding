# Plan 07-01: Runtime, Vault, and Main-Model Wiki Product Wiring

## Objective

Connect real terminal runtime sessions to one bound Vault and the production pinned-main-model Wiki workflow without weakening immutability, recovery, or secret boundaries.

## Tasks

1. Add stable project Vault binding resolution, sibling recommendation/default, explicit overrides, and finalized-session-aware runtime reopening.
2. Add runtime-owned finalization so active writer leases do not conflict; finalize deliberately on one-shot completion, interactive new/exit, and shutdown paths.
3. Implement the Provider-backed `WikiGenerationPort`, bounded evidence/current-page context builder, deterministic durability signals, and lifecycle orchestrator.
4. Add complete offline chain tests for no-op, synthesized, pending, wrong binding, crash/restart, and current-Wiki retrieval.
