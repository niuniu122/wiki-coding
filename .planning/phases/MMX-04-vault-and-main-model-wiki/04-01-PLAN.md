<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Project/Vault paths -> bootstrap | User-selected and externally editable paths must not escape or silently change project identity. |
| Active journal -> immutable raw | Truncation, corruption, sequence drift, and sensitive content must be rejected before evidence publication. |
| Prepared manifest/staging -> Wiki targets | Crash recovery and external edits must not cause blind replacement or rollback. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Spoofing | project manifest | critical | mitigate | canonical fingerprint, stable project ID, strict schema, mismatch rejection |
| T-04-02 | Tampering | raw journal/finalization | critical | mitigate | sequence/schema/hash validation and immutable terminal marker |
| T-04-03 | Information Disclosure | raw/Vault output | critical | mitigate | bounded sensitive-content denial and adversarial durable-output scan |
| T-04-04 | Elevation of Privilege | owned path resolution | critical | mitigate | fixed namespaces, relative path validation, lease, expected hashes |
| T-04-05 | Tampering | transaction recovery | critical | mitigate | synced prepared manifest, content hashes, ordered idempotent roll-forward |
| T-04-06 | Denial of Service | oversized records/pages | high | mitigate | finite byte/count limits and bounded recovery |
</threat_model>
