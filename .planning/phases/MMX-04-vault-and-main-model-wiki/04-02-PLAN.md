<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Human inbox -> immutable raw | Bytes, names, races, and unsupported formats are untrusted and human-owned until safely copied. |
| Raw/current context -> main model | Evidence may contain prompt injection; the model has no tools and may only propose a bounded patch. |
| Model patch -> core validator -> Vault | Fabricated sources, secrets, unsafe paths, stale hashes, and conflicting truth must stop before writes. |
| Workflow state -> restart | Pinned identity, generation attempts, usage, and commit outcome must not be duplicated or hidden. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-07 | Tampering | inbox import | high | mitigate | exact-byte hash, copy/sync/rename/re-hash, concurrent-change rejection |
| T-04-08 | Spoofing | source/model identity | critical | mitigate | allowed source inventory and immutable session binding with explicit rebind only |
| T-04-09 | Elevation of Privilege | model workflow | critical | mitigate | no tools, structured patch only, bounded context and reducer effects |
| T-04-10 | Information Disclosure | prompts/patches/receipts | critical | mitigate | sensitive marker rejection, source bounding, separate safe usage/status envelopes |
| T-04-11 | Tampering | current truth | critical | mitigate | expected hashes, one-current invariant, explicit supersession validation |
| T-04-12 | Repudiation | model cost/attempts | high | mitigate | durable job state, pinned identity, distinct usage and receipts |
</threat_model>
