<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Provider stream -> normalized tool call | Untrusted fragmented model output proposes effectful work and call identity. |
| Core effect -> journal/approval/tool ports | Ordering determines whether approval and side effects can be audited or replayed. |
| Journal -> restart recovery | Durable but possibly truncated/corrupt records reconstruct invocation truth. |
| Tool result -> Provider context | Only bounded, correctly associated, durable result data may influence the next model round. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-01 | Spoofing | ToolCallId/result association | critical | mitigate | validated exact ID, session uniqueness, full assistant call history, wrong-turn rejection |
| T-03-02 | Tampering | invocation journal ordering | critical | mitigate | pure legal-transition reducer, preview+append+sync, conflict failure |
| T-03-03 | Repudiation | approval decision | high | mitigate | durable per-call normalized request and explicit decision before started |
| T-03-04 | Elevation of Privilege | full-access | critical | mitigate | exactly two non-persisted modes; same preflight; unknown tools fail before decision |
| T-03-05 | Denial of Service | repeated tool loop/output | high | mitigate | finite round/call/time/result-byte budgets and serial execution |
| T-03-06 | Information Disclosure | tool result/history | high | mitigate | bounded structured result, raw fragment/stderr/reasoning exclusion |
| T-03-07 | Tampering | restart replay | critical | mitigate | post-start indeterminate terminal and no automatic replay |
</threat_model>
